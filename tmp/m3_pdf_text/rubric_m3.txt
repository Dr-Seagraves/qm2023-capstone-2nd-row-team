Component Breakdown
1. Model Specification (15 points)
Excellent (13-15 points):
✅ Model A (Fixed Effects): Correctly estimated with entity_effects=True and
time_effects=True
✅ Model B: Chosen specification (DiD/ARIMA/ML) is appropriate for research question
✅ Variable selection: Predictors are economically sensible (e.g., 2-month lag for FEDFUNDS based on
M2 EDA)
✅ Clustered standard errors: Used correctly (cov_type='clustered', cluster_entity=True)
✅ Code structure: Clear sections, comments explain logic, reproducible
Satisfactory (9-12 points):
✅ Both models estimated, but minor specification issues (e.g., forgot time FE, or used wrong lag)
✅ Variable selection reasonable but not optimal
✅ Standard errors computed but not clustered (heteroskedasticity concern)
✅ Code works but lacks clear comments
Needs Improvement (5-8 points):
⚠ One model estimated correctly, the other has errors
⚠ Variable selection is arbitrary (no justification from M2 or theory)
⚠ No robust standard errors (ignored heteroskedasticity)
⚠ Code is difficult to follow (poor structure)
Unsatisfactory (0-4 points):
❌ Neither model estimated correctly (wrong syntax, output errors)
❌ Models are nonsensical (e.g., time series regression on panel data)
❌ Code does not run

2. Diagnostics & Robustness (12 points)
Diagnostics (6 points):
Excellent (6 points):
✅ Heteroskedasticity test: Breusch-Pagan or visual inspection (residuals vs. fitted plot)

✅ Multicollinearity (VIF): Calculated for all predictors; high VIF addressed (variable dropped or
combined)
✅ Residual plots: Q-Q plot or histogram shows approximate normality (or fat tails acknowledged)
✅ All diagnostics interpreted: Not just reported, but explained (e.g., "VIF > 10 for SIZE, so we
dropped it")
Satisfactory (4-5 points):
✅ 2 of 3 diagnostics run (e.g., VIF + residual plot, but no heteroskedasticity test)
✅ Diagnostics reported but interpretation is weak
Needs Improvement (2-3 points):
⚠ Only 1 diagnostic run
⚠ Diagnostics run but not interpreted
Unsatisfactory (0-1 points):
❌ No diagnostics run

Robustness Checks (6 points):
Excellent (6 points):
✅ At least 3 robustness checks conducted:
Alternative lag structures (test lags 1, 2, 3)
Exclusion of outlier periods (e.g., COVID crash)
Sector subsamples (estimate model separately for sensitive vs. resilient sectors)
Placebo test (for DiD: run regression on pre-treatment period)
✅ Results compared: "Baseline model: β = -0.025. Excluding COVID: β = -0.023. Results are robust."
✅ Implications discussed: "Alternative lags yield similar coefficients, confirming lag 2 is not arbitrary."
Satisfactory (4-5 points):
✅ 2 robustness checks conducted
✅ Results compared but implications not fully discussed
Needs Improvement (2-3 points):
⚠ Only 1 robustness check
⚠ Checks conducted but results not compared
Unsatisfactory (0-1 points):
❌ No robustness checks

Frequently Asked Questions
Q: We only did 2 robustness checks instead of 3. Will we lose points?
A: Robustness Checks section = 6 points. With 2 checks (instead of 3), you'll receive 4-5/6 points (Satisfactory).
Lost: 1-2 points.

Q: Our DiD model failed the placebo test (found significant effect in pre-period). Is that okay?
A: No. A failed placebo test suggests parallel trends are violated, which undermines the causal interpretation
of your DiD estimate. Options:
1. Acknowledge this limitation in M3_interpretation.md and discuss why DiD may not be appropriate for
your data.
2. Choose a different Model B (ARIMA or ML comparison instead of DiD).
3. Restrict DiD to a subsample where parallel trends are more plausible.
Grading impact: If you acknowledge and address the violation, you'll receive partial credit. If you ignore it,
you'll lose points for Economic Reasoning.

Q: Can we use AI to help interpret our regression coefficients?
7/9

A: Yes, but you must verify that AI's interpretation is correct. Example:
AI output: "A 1% increase in FEDFUNDS reduces returns by 2.5%."
Your verification: "This phrasing is ambiguous. Correct: 'A 1 percentage point increase in FEDFUNDS
reduces returns by 2.5 percentage points.'"
Document this in AI_AUDIT_APPENDIX.md. If you don't verify and submit incorrect interpretations, you'll
lose points.

Q: We have heteroskedasticity but forgot to use clustered SEs. Can we resubmit?
A: No. Milestone grades are final. However, if you catch this before the deadline, you can fix it and resubmit
(no penalty if within deadline window).
Grading impact: Without clustered SEs, you'll lose points in Model Specification (down from 15/15 to
~11/15) and Diagnostics (down from 6/6 to ~4/6). Total lost: ~6 points.

Q: Our R² is 0.35. Will we lose points?
A: No. R² measures fit, not coefficient validity. What matters:
Are coefficients statistically significant? (t-stats, p-values)
Are they economically sensible? (signs, magnitudes)
Are they robust? (alternative specs yield similar results)
Low R² is common in financial data. Explain this in M3_interpretation.md: "While R² = 0.35 indicates that
35% of return variation is explained, our coefficient estimates are precise and robust. Financial returns are
inherently noisy due to unobserved factors (sentiment, liquidity), so moderate R² is expected."

Grading Philosophy
This rubric emphasizes:
1. Methodological rigor: Did you test assumptions and apply correct fixes?
2. Economic intuition: Do you understand WHY results make sense?
3. Honest self-assessment: Did you acknowledge limitations?
4. Reproducibility: Can someone else run your code and get identical results?
We are not grading:
Statistical sophistication (advanced models beyond course scope)
Perfect R² (unrealistic for financial data)
Length of code (concise is fine)
We are grading:
Did you apply course concepts correctly?
Can you defend your methodological choices?
Do you understand the economic implications of your results?
